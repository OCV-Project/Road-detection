
## Задача
Разработать программный модуль для автоматизированного выявления и классификации дефектов дорожного покрытия на основе анализа видеопотока с камеры мобильного комплекса или стационарной камеры наблюдения. Требуется реализовать:
1. Загрузка и анализ медиафайлов — поддержка видеофайлов (MP4, AVI, MKV), фотографий (JPG, PNG, BMP) и видеопотока с веб-камеры в реальном времени
2. Детекция дорожных дефектов — автоматическое обнаружение и классификация повреждений дорожного покрытия с использованием обученной нейросетевой модели YOLO11s на датасете RDD2022 (Road Damage Detection).
3. Классификация дефектов — разделение обнаруженных объектов на 9 категорий согласно стандарту RDD2022: продольные трещины (D00), поперечные трещины (D01), сетка трещин (D10), колея/выбоина (D11), яма (D20), заплатка (D40), размытая разметка (D43), размытая линия (D44), люк (D50), колейность (D0w0)
4. Количественная оценка — подсчёт общего числа обнаруженных дефектов каждого типа. Накопительная статистика по всем обработанным кадрам с вычислением распределения дефектов по классам
5. Настройка чувствительности — регулировка порога уверенности детекции в диапазоне 10–90% для адаптации к условиям освещения, качеству дорожного покрытия, погодным условиям и качеству видеопотока
6. Визуализация результатов — отображение ограничивающих рамок с цветовой индикацией по типам дефектов:

* Красный — критические дефекты (ямы, выбоины, сетка трещин)

* Оранжевый — трещины (продольные, поперечные)

* Жёлтый — колейность, размытая разметка

* Синий — заплатки, люки

7.  Подписи содержат код класса (D00–D50) и процент уверенности
8. Фильтрация ложных срабатываний — исключение из детекции объектов с аномальным соотношением сторон и площадью для подавления шумов на текстуре асфальта. Минимальный порог уверенности по умолчанию 25%
9. Экспорт отчёта — сохранение результатов анализа в формат CSV с группировкой по типам дефектов для последующей интеграции с системами управления дорожным хозяйством
10. Повторная обработка фото — функция обновления анализа фотографии при изменении порога уверенности без повторной загрузки файла.
11. Цветовая легенда — встроенная панель легенды с соответствием цветов и классов дефектов для оператора

## Методы решения
1. Детекция объектов на основе YOLO11s. Использована модель YOLO11s, дообученная на датасете RDD2022 (аннотации в формате XML Pascal VOC). Количество классов: 9 (D00–D0w0)
2. Предобработка кадра. Каждый кадр преобразуется в формат блоба размером 640×640 пикселей с нормализацией значений пикселей к диапазону [0, 1] и автоматической конвертацией цветового пространства BGR→RGB (параметр swapRB=true). Размер входного тензора: [1, 3, 640, 640]
3. Инференс и постобработка. Выходной тензор размерностью [1, 13, 8400] транспонируется в матрицу [8400, 13], где каждая строка содержит координаты ограничивающей рамки (cx, cy, w, h) и уверенности по 9 классам. Для каждой детекции выбирается класс с максимальной уверенностью. Применяется фильтрация по порогу уверенности (по умолчанию 25%) и Non-Maximum Suppression (NMS) с порогом перекрытия 0.45
4. Масштабирование координат. Координаты ограничивающих рамок масштабируются с размера входного тензора (640×640) до реального размера кадра с использованием коэффициентов scaleX и scaleY. Выполняется обрезка рамок по границам кадра для исключения выхода за пределы изображения
5. Группировка дефектов по категориям. Для удобства восприятия 9 классов объединены в 4 визуальные группы:
* Критические (D10, D20, D11) — красный цвет, требуют немедленного ремонта
* Трещины (D00, D01) — оранжевый цвет, требуют мониторинга
* Износ (D43, D44, D0w0) — жёлтый цвет, плановое обслуживание
* Инфраструктура (D40, D50) — синий цвет, люки и заплатки
6. Цветовая индикация. Для каждого обнаруженного дефекта определяется цвет рамки в соответствии с группой критичности. Подпись содержит код класса по стандарту RDD2022 и процент уверенности детекции
7. Статистический учёт. Для видеопотока ведётся подсчёт количества детекций каждого класса на каждом кадре. Статистика обновляется в реальном времени с отображением текущих значений по всем 9 классам. Для фотографий статистика формируется по единственному кадру. Предусмотрен сброс накопленной статистики и экспорт в CSV с группировкой по категориям
8. Архитектура с флагом активности. Обработка видеопотока реализована через рекурсивный вызов videoLoop() с использованием QTimer::singleShot(33, ...). Для гарантированной остановки воспроизведения используется флаг active, проверяемый в начале каждого цикла. Это исключает накопление отложенных вызовов в очереди событий Qt и обеспечивает мгновенную реакцию на нажатие кнопки «Стоп». Имя слота жёстко привязано к имени кнопки через Qt connectSlotsByName (on_stopBtn_clicked)

## Интерфейс GUI
Приложение реализовано в среде Qt Creator с использованием библиотеки OpenCV. Интерфейс включает:
Область отображения видео (QLabel) с автоматическим масштабированием кадра с сохранением пропорций и наложением рамок детекции с цветовой индикацией по группам критичности и подписями (код класса + уверенность в процентах). В верхнем левом углу кадра выводится сводная информация об общем количестве обнаруженных дефектов.
Панель управления (фиксированная ширина 220 пикселей) с кнопками:
* «Загрузить модель» — выбор ONNX файла
* «Фото» — загрузка изображения
* «Видео» — загрузка видеофайла
* «Камера» — захват с веб-камеры
* «Пауза/Продолжить» — приостановка анализа
* «Стоп» — полная остановка и освобождение ресурсов
* Информационную панель статистики (QTextEdit, только чтение) — отображение текущих значений количества дефектов каждого класса с группировкой по категориям критичности. Обновляется в реальном времени для видео, однократно для фото.
* Слайдер «Порог уверенности» (10–90%) с отображением текущего значения в формате «Порог: 0.25». Снабжён поясняющей подсказкой о влиянии на соотношение точности и полноты детекции. Изменение порога для фото приводит к немедленному повторному анализу без перезагрузки файла.
* Панель легенды — цветовые метки с названиями групп дефектов для быстрой интерпретации результатов оператором.
Кнопки управления статистикой: «Сброс» (обнуление всех счётчиков кадров и детекций), «CSV» (экспорт результатов в файл с временной меткой).
* Статусную строку с текущим состоянием системы («Готов», «Анализ...», «Остановлено», информация о загруженном файле).
